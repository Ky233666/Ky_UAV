# 路径规划地图与边界可视化说明

## 1. 功能定位

当前版本将 A*、RRT 和强化学习地图导出使用的规划区域统一到 `DroneManager` 维护的规划地图中。规划地图不是 Unity 场景本身，而是由场景中的无人机、任务点、起飞点和障碍物占地信息生成的二维占用网格。

该功能主要解决两个问题：

- 当任务点或障碍物超出默认规划边界时，A*/RRT 会直接返回“起点或终点超出规划边界”。
- 仅凭场景模型难以判断算法实际看到的障碍格子是否正确。

## 2. 关键代码

- `Assets/Scripts/UAV/Model/PlanningGridMap.cs`
  维护二维占用网格，提供世界坐标与网格坐标转换、障碍格查询和线段通行性检查。
- `Assets/Scripts/UAV/Controller/PlanningMapBuilder.cs`
  根据障碍物包围盒生成 `PlanningGridMap`，并按安全距离扩张障碍占地范围。
- `Assets/Scripts/UAV/Controller/DroneManager.cs`
  缓存规划地图，在规划参数、任务点、起飞点或障碍物变化后自动失效并重建。
- `Assets/Scripts/UAV/Controller/PlanningMapVisualizer.cs`
  在运行时显示规划边界和被占用的障碍格子。
- `Assets/Scripts/UAV/Comm/AStarPlanner.cs`
  A* 搜索时直接查询 `PlanningGridMap`，不再在搜索循环内反复做 Unity 物理检测。
- `Assets/Scripts/UAV/Comm/RRTPlanner.cs`
  RRT 采样、扩展和路径简化时使用同一份 `PlanningGridMap` 做碰撞判断。

## 3. 运行时操作

在 Play Mode 下打开右侧运行时控制面板，在“算法与规划”区域找到“规划地图”一行：

- `适配边界`
  根据当前无人机、任务点、起飞点和任务区域附近障碍物重新计算规划边界。
- `边界`
  显示或隐藏青色规划边界框。
- `障碍格`
  显示或隐藏红色占用格子，用于检查算法是否正确识别建筑物和自定义障碍物。

正常使用流程：

1. 放置或导入任务点。
2. 创建或调整自定义障碍物。
3. 点击 `适配边界`。
4. 打开 `边界` 和 `障碍格`，确认任务点、无人机和主要障碍物都位于青色边界内。
5. 切换到 A*、RRT 或 Q-learning 相关流程进行路径规划、导出或仿真。

## 4. 自动适配规则

`DroneManager.autoFitPlanningBoundsToScene` 默认为开启。系统在构建规划地图前会自动扩展规划边界，使当前无人机、任务点和起飞点位于规划区域内。

障碍物不会无限制扩大边界。系统只会纳入任务区域附近、且不会异常放大规划网格的障碍包围盒，避免城市大场景中某个巨大父级包围盒导致 A* 网格过大和卡顿。

相关参数：

- `planningGridCellSize`
  规划网格尺寸。越小越精细，但 A* 搜索节点更多。
- `planningObstacleSafetyPadding`
  障碍物占地额外安全距离。值越大路径越保守。
- `autoPlanningBoundsPadding`
  自动适配边界时在任务区域外额外保留的水平边距。
- `maxAutoPlanningBoundsSpan`
  过滤异常大障碍包围盒的水平跨度阈值。

## 5. 性能说明

规划地图只在必要时重建，A*/RRT 的搜索循环只查询缓存网格，因此任务点较多时不会再因为每个节点执行物理检测而明显卡顿。

障碍格可视化只用于调试和答辩展示。如果规划网格过大，系统会限制可视化数量，仍保留边界显示，避免生成过多临时方块影响运行。

算法过程演示默认只保留一架无人机的规划轨迹，并限制每条轨迹记录步数，避免多任务批量规划时积累大量搜索步骤。

## 6. 答辩表述

可以这样说明：

“系统中的 A* 和 RRT 并不是直接在连续三维场景中搜索，而是先将场景中的任务点、无人机和障碍物占地映射到二维规划网格。规划前系统会自动适配规划边界，并通过占用网格缓存障碍信息。这样既能减少 Unity 物理检测带来的运行开销，也能通过边界框和障碍格可视化检查算法实际使用的地图是否正确。”

需要注意的边界：

- 该模块仍属于二维路径规划，三维场景主要用于展示和碰撞告警。
- 多无人机任务由调度模块分配，路径规划仍是按单架无人机分别生成路径。
- 占用网格精度受 `planningGridCellSize` 影响，网格越粗，路径越可能偏保守。
