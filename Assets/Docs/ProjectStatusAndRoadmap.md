## 2. 项目定位

### 2.1 课题定位

本项目是一个基于 Unity 的多无人机协同巡检与路径规划仿真系统，目标不是单纯做一个“会飞的场景”，而是搭建一个可用于：

- 毕业设计展示
- 调度算法接入与对比
- 路径规划算法接入与对比
- 障碍环境下的路径可视化观察
- 多机任务执行演示
- 打包后的交互式展示

的完整仿真基础平台。

### 2.2 当前真实阶段

截至当前源码版本，项目已经从“原型场景”进入“最小可用演示系统”阶段，并且相较于早期版本，已经补齐了部分面向打包展示的运行时交互能力。

当前已经具备：

- 多无人机统一生成、统一管理、统一重置
- 调度算法切换与任务分配
- 路径规划算法切换与路径执行
- 规划路径与实际飞行轨迹可视化
- 任务点运行时添加、清空、导入
- 总览/跟随视角与多机观察
- 面向打包版的运行时控制面板

因此，当前版本已经不是“只能在 Inspector 里调参的内部原型”，而是具备较强答辩展示能力的第一版成型系统。

## 4. 当前系统结构

### 4.1 仿真总控

`SimulationManager.cs`

职责：

- 管理仿真状态：`Idle / Running / Paused`
- 绑定开始、暂停、重置按钮
- 在启动仿真时收集任务点并调用 `DroneManager` 进行任务分配
- 当场景中没有任务点时可自动导入默认任务点
- 运行时自动挂载 `SimulationRuntimeControlPanel`

现状判断：

- 已稳定可用
- 已成为当前场景运行流程的主入口

### 4.2 无人机管理

`DroneManager.cs`

职责：

- 统一生成和管理多架无人机
- 管理无人机数据、状态机和路径可视化组件
- 调用调度算法和路径规划算法
- 自动配置障碍层与建筑代理碰撞体
- 支持运行时重建机群、统一速度设置、统一轨迹可见性设置

当前支持的关键能力：

- `SpawnPoint` 起飞点优先生成
- `RespawnDrones(int count)`
- `ApplyDroneSpeedToAll(float newSpeed)`
- `ApplyPathVisibilityToAll(bool showPlannedPath, bool showTrail)`
- 调度算法枚举切换：`EvenSplit / GreedyNearest`
- 路径规划枚举切换：`StraightLine / AStar`
- 障碍层自动配置
- 建筑根节点自动递归设层
- 建筑代理 `BoxCollider` 自动生成

现状判断：

- 这是当前项目最核心的控制层之一
- 已经具备“运行时交互驱动”的能力，不再只是 Inspector 配置层

### 4.3 单机控制与状态执行

`DroneController.cs`

职责：

- 管理无人机基础运动
- 支持目标点与虚拟目标位置两种模式
- 支持回到初始位置与重置

`DroneStateMachine.cs`

职责：

- 管理任务执行状态流转
- 按任务队列推进任务
- 驱动无人机按 waypoint 路径逐点飞行

现状判断：

- 基础执行链路已经建立
- 当前仍偏演示型，不包含正式的多机冲突规避策略

### 4.4 路径可视化

`DronePathVisualizer.cs`

职责：

- 绘制当前规划路径
- 绘制实际飞行轨迹
- 支持分别开关规划路径和轨迹显示
- 在重置时清空并重新初始化轨迹

现状判断：

- 可视化效果已经能支撑课堂展示和算法对比
- 运行时面板已接入显示开关

### 4.5 任务点系统

`TaskPointUIManager.cs`

职责：

- 管理新增任务点、导入任务、清空任务按钮
- 支持点击场景放置任务点
- 支持落点合法性检测
- 支持鼠标落点预览圈
- 支持 `Esc` 取消放置模式
- 放置时自动切换到总览视角，便于交互

`TaskPointSpawner.cs`

职责：

- 运行时创建任务点
- 基于障碍物检测自动避开建筑和阻挡区域
- 支持不安全位置重试定位

`TaskPointImporter.cs`

职责：

- 从资源中导入任务点数据

现状判断：

- 当前任务点交互已比较完整
- 对答辩演示来说，交互性明显高于纯静态导入方案

### 4.6 相机系统

`CameraManager.cs`

职责：

- 管理总览相机与跟随相机
- 在多机之间切换跟随目标
- 支持总览相机自由移动、旋转和缩放

当前键位：

- `1`：总览视角
- `2`：跟随视角
- `Tab / E`：下一架无人机
- `Q`：上一架无人机
- `W/A/S/D`：总览平移
- `R / F`：升降
- 鼠标滚轮：缩放
- 右键拖动：旋转
- `Shift`：加速

现状判断：

- 已满足演示、观察和调试需求

### 4.7 主界面样式与运行时控制面板

`SimulationDashboardStyler.cs`

职责：

- 统一主界面的标题卡、状态卡、底部控制区和任务区样式
- 清理旧版残留右侧栏，避免与新面板重叠

`SimulationRuntimeControlPanel.cs`

职责：

- 在运行时自动创建右侧控制面板
- 支持展开/收起
- 支持滚动区域
- 支持摘要、状态提示和交互控制
- 使用更紧凑的布局，避免压住底部任务工具区

当前已接入的运行时功能：

- 调度算法切换
- 路径规划算法切换
- 无人机数量调整
- 飞行速度调整
- 仿真倍速调整
- 规划路径显示开关
- 飞行轨迹显示开关
- 总览 / 跟随 / 下一架镜头控制
- 同步当前机群设置
- 重建机群

现状判断：

- 右侧运行时面板已正式落地
- 它已经替代了部分过去只能在 Inspector 中修改的关键参数
- 这是当前版本相对于旧版最重要的一次交互升级

## 6. 当前仍未真正完成的部分

### 6.1 多机冲突规避

当前情况：

- 项目已有多机调度和路径执行
- 但还没有形成正式的多机冲突规避策略

影响：

- 当路径接近、交叉或重叠时，仍可能缺少可解释的协同避让机制

### 6.2 实时统计面板

当前情况：

- 运行时面板现在主要负责控制，不负责完整统计
- 目前只显示摘要信息，不是正式实验统计面板

缺口：

- 总任务数、完成数
- 各无人机状态与完成数
- 飞行距离
- 总耗时
- 冲突次数
- 等待次数

### 6.3 实验结果导出

当前情况：

- 没有正式的 CSV 或 JSON 实验结果导出功能

这意味着：

- 可以演示
- 但还不方便做论文中的实验记录、表格和对比图

### 6.4 更强的算法对照组

当前情况：

- 现在已有一组调度对比和一组路径规划对比
- 但整体说服力仍偏“最小可用”

后续更优方向：

- 再补一类调度算法
或
- 再补一类路径规划算法

### 6.5 Inspector 参数迁移尚未完全结束

当前情况：

- 关键参数已经迁移了一部分到运行时面板
- 但规划网格、边界、对角搜索、障碍配置等仍主要依赖 Inspector

判断：

- “运行时控制”已经开始落地
- 但还没有完全覆盖实验级参数

## 8. 当前最推荐的开发顺序

### P0：当前已经完成

以下内容可以视为当前版本已完成：

1. 调度算法接入
2. 路径规划算法接入
3. 路径执行与轨迹显示
4. 任务点点击放置
5. 基础相机交互
6. 运行时控制面板第一版

### P1：现在最值得继续做

#### 任务 1：补实时统计面板

建议优先显示：

- 当前调度算法
- 当前路径规划算法
- 总任务数 / 已完成任务数
- 无人机数量
- 每架无人机状态
- 每架无人机飞行距离
- 当前跟随目标
- 仿真总耗时

这是当前最缺的一块，因为它能把系统从“能看飞行”推进到“能看结果”。

#### 任务 2：补实验结果导出

建议优先做：

- 单次实验结果结构体
- CSV 导出
- 仿真结束后导出一行实验记录

这是论文实验闭环最关键的补充。

#### 任务 3：补多机冲突规避策略

建议至少实现一种简单且可解释的策略，例如：

- 同路段等待
- 近距离让行
- 局部暂停后重试

不要一步做复杂，但要让系统对“多机协同”更有说服力。

#### 任务 4：继续扩展运行时控制面板

建议下一批迁移的参数：

- `allowDiagonalPlanning`
- `planningGridCellSize`
- `planningWorldMin`
- `planningWorldMax`
- 障碍自动配置开关
- 相机跟随偏移

### P2：有余力再做

- 第二种更强的调度算法
- 第二种更强的路径规划算法
- 批量实验
- 多组任务集切换
- 统计图表面板
- 更真实的飞行物理或能耗模型

## 10. 后续协作默认规则

后续如果继续让 AI 或其他协作者接手，默认遵循以下规则：

1. 先阅读本文件和 `README.md`。
2. 先核对主场景 `Assets/Scenes/Main/MainScene.unity` 的实际运行结果，再做结构性修改。
3. 若增加算法，必须同步补充：
   - 算法解决什么问题
   - 输入输出是什么
   - 如何接入当前控制层
   - 在 Unity 中如何验证
4. 若新增运行时交互，优先考虑打包版可用性，不要只面向 Editor。
5. 每完成一个核心模块，都要同步更新本文档，避免路线图再次过期。

- 多无人机统一管理
