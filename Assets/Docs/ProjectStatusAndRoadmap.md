# 项目进展与后续开发路线

## 1. 文档用途

这份文档是当前项目的唯一总说明文档，用来回答四件事：

1. 这个项目现在已经做到什么程度。
2. 哪些功能已经真正完成，哪些只是有骨架。
3. 目前最值得继续补的功能是什么。
4. 后续如果继续让 AI 或协作者接手，应该按什么顺序推进。

## 2. 项目定位

### 2.1 课题名称

基于 Unity 的多无人机协同路径规划与调度算法仿真系统设计与实现

### 2.2 项目目标

本项目最终目标不是单纯“让几架无人机飞起来”，而是形成一个可以用于本科毕设展示、论文实验和结果对比的仿真系统。系统应具备以下能力：

1. 支持多无人机执行多任务点巡检。
2. 支持任务调度算法切换与对比。
3. 支持路径规划算法切换与对比。
4. 支持静态障碍环境下的路径规划与避障展示。
5. 支持路径、轨迹、状态、统计指标的可视化。
6. 支持实验结果汇总与导出。

### 2.3 当前真实定位

截至当前版本，项目已经从“基础原型”进入“最小可用算法演示版”阶段。

更准确地说：

- 平台骨架已经完成。
- 第一版调度算法已经接入。
- 第一版路径规划算法已经接入。
- 无人机已经可以按规划路径点飞行，而不是只会直线冲向目标。
- 路径和飞行轨迹已经可以在场景中直接看到。

但它还不是完整的毕设成品，因为以下关键部分仍未完成：

- 障碍环境的正式建模和稳定绕障演示闭环
- 多机冲突规避
- 实时指标统计面板
- 实验结果导出
- 第二组对比算法

一句话总结：

当前项目已经不再只是“演示型无人机场景”，而是已经具备了“毕设核心算法闭环的第一版”，但实验层和数据层还明显不足。

## 3. 当前系统结构

### 3.1 运行环境

- Unity 版本：`2022.3.62f3`
- 开发语言：`C#`
- 当前主场景：`Assets/Scenes/Main/MainScene.unity`
- 当前主要脚本目录：`Assets/Scripts/UAV/Controller`

### 3.2 核心模块概览

- `SimulationManager.cs`
  - 管理仿真状态：`Idle / Running / Paused`
  - 绑定开始、暂停、重置按钮
  - 开始仿真时自动收集任务点并触发任务分配
  - 当场景中没有任务点时，可自动从 `Resources/taskpoints.csv` 导入默认任务点

- `DroneManager.cs`
  - 批量生成和重置无人机
  - 维护 `DroneController` 与 `DroneData`
  - 调用任务调度算法分配任务
  - 调用路径规划算法为当前任务规划路径
  - 为每架无人机关联状态机和路径可视化组件

- `DroneController.cs`
  - 管理无人机基础运动控制
  - 当前采用“脚本驱动位置”的非物理推进模式
  - 已修复进入 `Play` 后因刚体物理导致的乱飞问题

- `DroneStateMachine.cs`
  - 管理无人机状态：`Idle / Moving / Waiting / Finished`
  - 按任务队列逐个执行任务
  - 已支持按照规划路径中的 waypoint 逐点飞行
  - `Waiting` 仍主要是预留状态，尚未形成完整冲突规避逻辑

- `DroneData.cs`
  - 存储无人机编号、任务队列、状态、完成任务数、等待原因
  - 已增加路径规划相关数据：
    - `plannedPath`
    - `currentWaypointIndex`
    - `currentPlannerName`

- `TaskPoint.cs`
  - 管理任务点状态：`Pending / InProgress / Completed`
  - 支持任务点颜色显示

- `TaskManager.cs`
  - 管理场景中的任务点集合
  - 支持获取待执行任务和优先级排序

- `TaskPointSpawner.cs`
  - 运行时创建任务点

- `TaskPointImporter.cs`
  - 从 `Resources` 中导入 CSV / JSON 任务点

- `CameraManager.cs`
  - 管理总览视角与跟随视角
  - 已支持运行时自动绑定无人机
  - 已支持切换跟随目标无人机

- `SimulationDashboardStyler.cs`
  - 负责主界面“指挥台风格”UI 自动整理与样式统一

- `DronePathVisualizer.cs`
  - 显示当前规划路径
  - 显示飞行轨迹

## 4. 当前已经完成的内容

## 4.1 仿真平台基础骨架

已完成：

- 主场景、地面环境、建筑、无人机、任务点、相机、UI 的基础搭建
- 仿真开始、暂停、重置的基本控制链路
- 多无人机统一生成和管理
- 主界面基础样式优化

当前结论：

- 已达到“可以稳定运行并进行演示”的程度
- 不再是只有静态场景和零散脚本的半成品

## 4.2 多无人机运行与场景稳定性修复

已完成：

- 统一由 `DroneManager` 生成受控无人机
- 清理了主场景中遗留的旧版单机对象问题
- 修复了“场景里显示 5 架，但只有 4 架受控”的问题
- 修复了点击 `Play` 后无人机因物理刚体而乱飞的问题
- 开始、暂停、重置目前已能控制受管无人机

当前结论：

- 多机基础运行闭环已经稳定
- 当前 5 架无人机可以统一响应仿真控制

## 4.3 任务系统

已完成：

- 场景内任务点识别
- 运行时手动新增任务点
- 清空任务点
- CSV / JSON 导入任务点
- 当场景中没有任务点时，自动导入默认任务点数据

当前限制：

- 任务点生命周期管理还不够完整
- 缺少实验任务集保存、批量生成规则、任务集切换方案

## 4.4 算法接口层

已完成：

- `IPathPlanner`
- `ISchedulerAlgorithm`
- `PathPlanningRequest / PathPlanningResult`
- `SchedulingRequest / SchedulingResult`
- 调度算法类型枚举
- 路径规划算法类型枚举

当前结论：

- 算法层已经和控制层初步解耦
- 后续新增算法时，不需要再把逻辑硬写进 `DroneManager` 或 `SimulationManager`

这部分可以视为 `P0 / 任务 1` 已完成。

## 4.5 调度算法

已完成：

- `EvenSplitScheduler`
  - 用作基础对照组
  - 将任务点按数量均分给各无人机

- `GreedyNearestScheduler`
  - 作为当前第一版真实调度算法
  - 基于“最近优先”，并可考虑任务优先级权重

当前结论：

- 任务分配已经不再只有简单均分这一条路
- 已经具备最基础的“调度算法可切换、可对比”能力

当前不足：

- 还没有更复杂的调度算法作为第二对照组
- 没有形成实验统计输出，暂时只能通过场景表现观察差异

这部分可以视为 `P0 / 任务 2` 已完成。

## 4.6 路径规划算法

已完成：

- `StraightLinePlanner`
  - 作为最基础保底规划器

- `AStarPlanner`
  - 当前第一版真实路径规划算法
  - 基于 `XZ` 平面的二维网格搜索
  - 支持四方向 / 八方向搜索
  - 支持基于障碍层的网格阻挡检测
  - 输出 waypoint 路径点列表

当前结论：

- 项目已经具备最基础的“路径规划算法切换”能力
- 系统已经不是“所有无人机只会直线飞向目标”的状态

当前不足：

- 还缺少第二种路径规划算法用于对比
- 静态障碍层虽然已接入到 A*，但还需要场景配置和展示层进一步完善

这部分可以视为 `P0 / 任务 3` 已基本完成。

## 4.7 路径执行与可视化

已完成：

- 无人机按 `plannedPath` 的 waypoint 逐点飞行
- 不再只取规划终点进行直线移动
- 规划路径可视化
- 实际飞行轨迹可视化

当前结论：

- 用户已经可以直接在场景中观察“计划路径”和“真实飞行轨迹”
- 这一步让 A* 与直线飞行之间的差异具备了可观察性

当前不足：

- 还没有把搜索网格、障碍检测范围、关键路径节点等更细的算法信息显示出来
- 还没有在 UI 上展示当前使用的算法名称和路径长度等信息

这部分可以视为 `P1 / 任务 5` 已完成。

## 4.8 相机与观察视角

已完成：

- `1` 键切换总览视角
- `2` 键切换跟随视角
- `Tab / E` 切换到下一架无人机
- `Q` 切换到上一架无人机
- 跟随相机已支持平滑跟随
- 跟随目标会自动从当前受管无人机列表中获取

当前结论：

- 视角系统已经比最初版本更适合做演示和答辩展示
- 观察多机行为和不同路径效果已经方便很多

## 4.9 UI 与交互

已完成：

- 主控制按钮
- 任务工具按钮
- 状态文本
- 主界面整体风格改造
- 修复了 `SimulationDashboardStyler` 在编辑器阶段触发的 `SendMessage cannot be called during Awake/OnValidate` 警告

当前不足：

- 缺少算法选择面板
- 缺少路径规划参数和调度参数可视化操作区
- 缺少实时统计数据面板

## 5. 当前最真实的完成度判断

### 5.1 已经可以认为完成的层

- 场景基础层
- 仿真控制层
- 多无人机生成与基础执行层
- 算法接口层
- 第一版调度算法层
- 第一版路径规划算法层
- 路径执行与轨迹可视化层
- 相机观察层

### 5.2 仍然只算“第一版骨架”的层

- 静态障碍环境正式演示层
- 多机冲突规避层
- 实时统计层
- 实验结果导出层
- 对比实验层

### 5.3 当前项目最缺的已经不是“能不能跑”，而是“能不能做实验”

当前项目已经具备：

- 可以展示
- 可以演示
- 可以说明已经有算法接入

但还不具备：

- 足够完整的实验指标
- 足够规范的对比输出
- 足够稳定的障碍规避与冲突处理说明

因此现在最影响毕设质量的部分，不再是基础运动和基础 UI，而是实验与数据链路。

## 6. 当前没有完成但毕设必须补上的内容

## 6.1 静态障碍环境闭环

虽然 `AStarPlanner` 已经支持基于障碍层进行阻挡检测，但仍需要把这件事补成完整闭环：

- 统一场景障碍物层
- 为障碍物设置稳定的碰撞边界与规划边界
- 调整规划区域和网格参数
- 明确做出“有障碍时会绕开、无障碍时走更短路径”的演示场景

最低要求：

- 在主场景中可以稳定复现绕障效果
- 不需要靠“碰运气”才能看到 A* 的优势

## 6.2 多机冲突规避

当前 `Waiting` 状态仍然主要是预留状态，尚未形成完整的多机协调机制。

必须补上至少一种简单可解释的策略，例如：

- 同路径点等待
- 近距离让行
- 局部重规划

最低要求：

- 两架无人机不会长时间占用同一段路径导致明显卡死

## 6.3 实时统计面板

当前 `DroneData` 已经有部分统计字段，但没有形成统一展示。

必须补上：

- 总任务数
- 已完成任务数
- 每架无人机完成任务数
- 每架无人机飞行距离
- 仿真总耗时
- 等待次数
- 冲突次数

最低要求：

- 在 Unity 界面中可以实时看到关键指标变化

## 6.4 实验数据导出

这是毕设论文非常关键的一环，目前还没有开始。

必须补上：

- 实验结果汇总结构
- CSV 导出
- 一次仿真结束后的结果落盘

最低要求：

- 至少能导出一行完整实验记录，供论文表格和图表使用

## 6.5 第二组对比算法

当前已经有：

- 调度：`EvenSplit` vs `GreedyNearest`
- 路径规划：`StraightLine` vs `AStar`

这已经能做“最基础对照”，但毕设说服力还不够强。

建议后续至少再补一项：

- 第二种调度算法
  或
- 第二种路径规划算法

优先建议：

- 路径规划补 `RRT` 或简化版 `RRT`
  或
- 调度补一个带代价函数的增强贪心 / 遗传算法简化版

## 8. 当前建议的下一步

如果继续按毕设主线推进，下一步最推荐做的是：

1. 完善障碍层和场景障碍配置，让 A* 绕障效果稳定可见。
2. 补一个实时统计面板，把当前算法名、任务进度、无人机状态和飞行距离显示到 UI。
3. 在此基础上，再做实验数据导出。

不建议当前优先做的事情：

- 继续投入大量时间做纯美化 UI
- 一上来就写特别复杂的高级算法
- 大范围重构目录结构

原因很简单：

当前最缺的是“实验闭环”，不是“外观更炫”。

## 9. 毕设最小可交付版本判断

如果能把下面几项做完，就已经具备较完整的本科毕设交付能力：

1. 多无人机正常运行，任务点可导入，仿真可开始/暂停/重置。
2. 具备至少两种调度策略对照。
3. 具备至少两种路径规划策略对照。
4. A* 在障碍环境中可稳定演示绕障。
5. 无人机按规划路径飞行，并能显示规划路径和实际轨迹。
6. UI 中可实时显示核心统计指标。
7. 可导出实验结果 CSV。

截至当前版本，这 7 条里已经完成了前 5 条的大部分框架，但第 4 条还需要场景层完善，第 6 条和第 7 条还没有开始。

## 10. 后续协作默认规则

以后如果继续让 AI 或协作者接手本项目，默认按以下规则推进：

1. 先读本文件。
2. 优先做 `P1` 和 `P2` 中尚未完成的内容。
3. 每完成一个关键功能，都要同步更新本文件。
4. 每新增一个算法，都要明确写清：
   - 它解决什么问题
   - 输入输出是什么
   - 如何与当前系统集成
   - 在 Unity 里如何验证它已经生效

## 11. 当前结论

当前项目已经完成了从“原型场景”到“最小可用算法演示系统”的关键跃迁。

已经真正落地的核心进展是：

- 调度接口层完成
- 路径规划接口层完成
- 贪心调度完成
- A* 路径规划完成
- 无人机按 waypoint 路径执行完成
- 路径与轨迹可视化完成
- 视角切换与多机跟随观察完成

现在最该补的，不再是“无人机能不能飞”，而是：

- 障碍环境闭环
- 冲突规避
- 实时统计
- 实验导出

只要把这四块补齐，这个项目就能更自然地从“演示系统”升级为“真正能支撑本科毕设答辩和论文实验分析的仿真系统”。
